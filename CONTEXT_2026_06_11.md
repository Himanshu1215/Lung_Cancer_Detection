# Session Context — 2026-06-11

## What this project is
Lung cancer detection study using **LUNA16** CT scan data.
The core research question is **data leakage**: comparing Protocol A (scan-level split, leaky) vs Protocol B (patient-level split, clean) to measure how much leakage inflates model performance.

---

## What was completed today

### 1. Preprocessing pipeline — DONE
- Script: `src/data/lidc_prepare.py`
- Processed all 10 LUNA16 subsets (subset0–subset9) into 224×224 PNG patches
- HU window: center=-600, width=1500 → uint8
- 64mm physical crop per nodule, 3 slices per nodule (iz-1, iz, iz+1), 5 normal slices per scan
- Output: `data/luna16/processed/subsetN/*.png` + per-subset manifest CSVs
- Took ~10 min total (slow because data lives on network filesystem `/mnt/sfs`)
- Idempotent: `.done` sentinel files skip already-processed scans on re-run

### 2. Splits — DONE

- Protocol B (clean): subsets 0-4 → train, 5 → val, 6 → test. No scan appears in multiple splits.

## Why preprocessing was slow (for reference)
Each CT scan is a 100–300 MB `.mhd`+`.raw` 3D volume read from `/mnt/sfs` (network filesystem).
Processing is single-threaded. 10 min for ~890 scans is expected. Re-runs are instant (`.done` sentinels).
To speed up: add `multiprocessing.Pool` over `mhd_files` in `lidc_prepare.py:335`.
